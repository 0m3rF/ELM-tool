## v1 Requirements (Shipped)

### Integration and Launch

- [x] **BOOT-01**: GUI automatically launches via `CustomTkinter` when the user invokes the main CLI entrypoint without any arguments.
  - *Outcome:* `elm.py:main()` checks `len(sys.argv) == 1` and calls `launch()` → `ELMApp()`
- [x] **BOOT-02**: GUI terminates completely when the user closes the root window.
  - *Outcome:* `ELMApp._on_close()` calls `self.destroy()` + `sys.exit(0)`

### Environment Management

- [x] **ENV-01**: User can view a list of all configured environments via the GUI.
  - *Outcome:* `EnvironmentListPanel.refresh()` calls `api.list_environments(show_all=True)` and renders scrollable buttons
- [x] **ENV-02**: User can create a new environment via a visual form.
  - *Outcome:* `EnvironmentFormPanel` create mode → `api.create_environment(...)` with full validation
- [x] **ENV-03**: User can edit an existing environment's credentials/details via a visual form.
  - *Outcome:* `EnvironmentFormPanel` edit mode → `api.update_environment(...)` with partial field updates
- [x] **ENV-04**: User can delete an existing environment via the GUI.
  - *Outcome:* Delete button → `messagebox.askyesno` → `api.delete_environment(...)`

### Task Execution (Copy)

- [x] **EXEC-01**: User can select a source environment and target environment from dropdowns to prepare a copy task.
  - *Outcome:* `OperationsPanel` has two `CTkOptionMenu` dropdowns populated from `api.list_environments()`
- [x] **EXEC-02**: User can trigger the copy/mask logic by clicking a "Run" or "Execute" button.
  - *Outcome:* Execute button → `_on_execute()` validates fields → spawns worker thread → calls `api.copy_db_to_db(...)`
- [x] **EXEC-03**: The execution of backend tasks must run in a background thread and not freeze the CustomTkinter UI.
  - *Outcome:* `threading.Thread(daemon=True)` with `queue.Queue` for stdout redirect; main loop never blocks

### Observability

- [x] **MON-01**: A persistent log panel exists at the bottom or side of the GUI.
  - *Outcome:* `OperationsPanel` bottom `CTkFrame` with `CTkTextbox(state="disabled")` log display
- [x] **MON-02**: Standard output (`stdout`) and standard error (`stderr`) from the core CLI execution are redirected to and rendered in the log panel in real-time.
  - *Outcome:* `QueueStream` redirects `sys.stdout`/`sys.stderr` to `queue.Queue`; main thread drains every 100ms into log textbox
