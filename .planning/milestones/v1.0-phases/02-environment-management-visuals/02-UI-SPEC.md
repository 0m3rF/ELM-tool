# Phase 2 — UI Design Contract

> Visual and interaction contract for the Environment Management Visuals phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | CustomTkinter (native Python desktop) |
| Preset | `dark-blue` (CTk built-in theme) |
| Component library | CTk native widgets (`CTkFrame`, `CTkEntry`, `CTkButton`, `CTkOptionMenu`, `CTkCheckBox`, `CTkScrollableFrame`, `CTkLabel`) |
| Icon library | Unicode glyphs (no external dependency) |
| Font | System default (CTk inherits OS font) — no custom font loading required |

---

## Spacing Scale

Declared values (must be multiples of 4, applied via `padx`/`pady`/`width` kwargs):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline element padding |
| sm | 8px | Compact element spacing, intra-form gaps |
| md | 16px | Default element spacing, form field vertical gaps |
| lg | 24px | Section padding inside panels |
| xl | 32px | Gap between left panel and right panel |

Exceptions: Left panel fixed width at `220px` (not a spacing token — layout dimension per D-01).

---

## Typography

CustomTkinter uses `font=(family, size, weight)` tuples. System font is inherited.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | normal (400) | CTk default | Form field labels, list item text, status messages |
| Label | 12px | normal (400) | CTk default | Section headers ("Environments"), helper text, placeholder text |
| Heading | 18px | bold (700) | CTk default | Panel titles ("Environment Details"), empty state heading |

**Font declarations in code:**
- Body: `font=("", 14)`
- Label: `font=("", 12)`
- Heading: `font=("", 18, "bold")`

---

## Color

CustomTkinter `dark-blue` theme provides the color palette. Colors are specified via CTk widget kwargs, not raw hex values, to ensure light/dark mode compatibility.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | CTk theme default (`fg_color` of root/frames) | Main background, form panel surface, tab content area |
| Secondary (30%) | `CTkFrame` default surface | Left list panel background, form group containers |
| Accent (10%) | CTk theme accent (dark-blue) | Selected environment highlight, "Save Environment" button, "Test Connection" button |
| Destructive | `#DC3545` (red) | "Delete" button `fg_color`, validation error label `text_color` |
| Success | `#28A745` (green) | "✓ Connected" status label `text_color` |

Accent reserved for:
- Selected item highlight in the environment list (`fg_color` on the selected `CTkButton`)
- Primary action buttons: "Save Environment", "Test Connection", "New Environment"
- Never used for labels, borders, or passive elements

## Interaction States

### Form Mode State Machine

| Action | From | To | Form Behavior |
|--------|------|----|---------------|
| Click "＋ New" | idle | create | Clear all fields, enable name field, hide delete/test buttons |
| Click list item | idle/create | edit | Populate fields (except password per D-02), disable name field, show all buttons |
| Click "Save" (success) | create/edit | idle | Clear form, refresh list, deselect |
| Click "Discard Changes" | create/edit | idle | Clear form, deselect list item |
| Click "Delete" (confirmed) | edit | idle | Delete env, refresh list, clear form |

### Empty State (No Environments)

When `api.list_environments()` returns an empty list:
- Left panel: show empty scrollable area (no items)
- Right panel: display the empty state content (see Copywriting Contract below)

### Loading Behavior

- Environment list: loaded synchronously on widget init and after each mutation (create/update/delete)
- Connection test (D-03): synchronous call — button text changes to "Testing..." and `state="disabled"` during execution. Result shown in status label.

## Form Validation Rules

Batch validation on submit (not real-time per-field):

| Rule | Create Mode | Edit Mode |
|------|-------------|-----------|
| `name` non-empty and not `"*"` | Required | N/A (disabled) |
| `host` non-empty | Required | Optional (skip if empty) |
| `port` valid integer | Required | Optional (skip if empty) |
| `user` non-empty | Required | Optional (skip if empty) |
| `password` non-empty | Required | Optional (skip if empty — means "no change") |
| `service` non-empty | Required | Optional (skip if empty) |
| `db_type` selected | Required | Optional |
| If `encrypt` checked → `encryption_key` non-empty | Required | Required |
| If `db_type` = ORACLE → `connection_type` selected | Required | Required |

## Data Flow Contract

```
User action → EnvironmentFormPanel method
  → calls elm.api.{create,update,delete,test}_environment()
  → on success: EnvironmentManagerFrame.refresh_list()
  → on error: EnvironmentFormPanel._show_error(message)
```

**API functions used by this phase:**

| API Function | Trigger | Response Handling |
|-------------|---------|-------------------|
| `api.list_environments(show_all=True)` | Widget init, after any mutation | Populate left panel list items |
| `api.get_environment(name, encryption_key)` | Click list item (for encrypted envs) | Populate form fields (except password) |
| `api.create_environment(...)` | Click "Save" in create mode | Refresh list, show success, clear form |
| `api.update_environment(...)` | Click "Save" in edit mode | Refresh list, show success |
| `api.delete_environment(name)` | Confirm delete dialog | Refresh list, clear form, show success |
| `api.test_environment(name, encryption_key)` | Click "Test Connection" (D-03) | Show ✓/✗ status label |

## Conditional Visibility Rules

| Condition | Elements Affected | Behavior |
|-----------|-------------------|----------|

| Mode == `create` | Delete button, Test Connection button | Hidden |
