```python
@contextlib.contextmanager
def file_lock(file_path: str, timeout: float = 10.0):
    lock_file = f"{file_path}.lock"
    lock = FileLock(lock_file, timeout=timeout)
    try:
        with lock:
            yield
    finally:
        pass
```

Create `elm/core/history.py` implementing `HistoryRecorder` per D-01 through D-14 from CONTEXT.md:

1. **HistoryRecord dataclass** (D-06): `id`, `date`, `operation_type`, `source`, `target`, `query`, `table`, `mode`, `batch_size`, `parallel_workers`, `start_time`, `end_time`, `status`, `record_count`, `error_message`. All fields typed with Optional where nullable. `id` is int, `status` is Literal["success", "failure"].

2. **HistoryRecorder class**:
   - `__init__(self, config_manager=None)` — uses `get_config_manager()` default.
   - `record(self, result: OperationResult, params: Dict[str, Any]) -> bool` — the public API.
   - Private helpers: `_read_records(history_file)`, `_write_records(history_file, records)`, `_create_backup(file_path)`, `_verify_and_cleanup(file_path, backup_path)`, `_restore_from_backup(file_path, backup_path)`, `_cleanup_old_backups(history_file)`.

3. **record() logic** (D-07, D-08, D-10 through D-13):
   - Get `history_file = self.config.get_history_file()` and `max_entries = self.config.get_config_value("history.max_entries") or 100`.
   - Acquire `file_lock(history_file)`.
   - Read existing records via `_read_records()` (returns `[]` if file missing).
   - Compute next id: `max([r.id for r in records], default=0) + 1`.
   - Build `HistoryRecord` from `result` and `params`. Set `date = datetime.now().isoformat()`. Map `result.success` to `status`.
   - Append record, trim: `records = records[-max_entries:]`.
   - Create backup via `_create_backup()` (shutil.copy2 if file exists).
   - Write records via `_write_records()` (json.dump, indent=2, ensure_ascii=False).
   - Verify via `_verify_and_cleanup()` (open and json.load; on success, delete .bak).
   - On ANY exception during write/verify: catch, call `_restore_from_backup()`, return False.
   - On success: return True.
   - On exception at top level (e.g., missing config): return False.

4. **params mapping** from copy function locals to HistoryRecord fields:
   - `operation_type`: derived from caller ("db2db", "db2file", "file2db")
   - `source`: `params.get("source_env")`
   - `target`: `params.get("target_env")` or `params.get("file_path")`
   - `query`: `params.get("query")`
   - `table`: `params.get("table")`
   - `mode`: `params.get("mode")`
   - `batch_size`: `params.get("batch_size")`
   - `parallel_workers`: `params.get("parallel_workers")`
   - `start_time`: `params.get("start_time")` or fallback to `datetime.now().isoformat()`
   - `end_time`: `params.get("end_time")` or fallback to `datetime.now().isoformat()`
   - `record_count`: `result.record_count`
   - `error_message`: `result.error_details` if not success else None

5. **Backup cleanup** (from RESEARCH.md Open Questions): `_cleanup_old_backups()` deletes `.bak` files older than 24h in the same directory. Call once per `__init__`.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | history.json write path | mitigate | Path is resolved through ConfigManager.get_history_file() which uses get_elm_tool_home() — user-controlled but no external input path traversal |
| T-04-02 | Tampering | JSON corruption on crash | mitigate | Backup-before-write with verification (D-10 through D-13) |
| T-04-03 | Denial of Service | Unbounded file growth | mitigate | FIFO eviction at max_entries (D-08, D-09) |
| T-04-04 | Information Disclosure | .bak file left behind | mitigate | Cleanup on success + _cleanup_old_backups() on init |
</threat_model>
